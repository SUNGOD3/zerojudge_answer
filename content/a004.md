# a004 - 文文的求婚

[🔗 前往 ZeroJudge 原題](https://zerojudge.tw/ShowProblem?problemid=a004)

**🏷️ 標籤**: `Conditional Logic`, `Arithmetic`

## 題目描述
本題要求判斷給定的西元年份是否為閏年。輸入會有多行，每行包含一個整數代表年份，直到檔案結束 (EOF)。對於每個輸入的年份，程式需輸出 "閏年" 或 "平年"。題目中明確定義了閏年的規則：西元年份若能被 4 整除且不被 100 整除，或者能被 400 整除，則為閏年。

## 解題思路
本題的核心是根據題目提供的閏年規則進行判斷。閏年的判斷條件可以歸納為以下兩種情況之一：
1.  年份能被 400 整除。
2.  年份能被 4 整除，但不能被 100 整除。

只要滿足這兩個條件中的任何一個，該年份就是閏年；否則，為平年。

提供的 C++ 程式碼採用了兩階段的 `if` 判斷：
1.  首先，檢查年份 `Y` 是否能被 400 整除 (`Y%400==0`)。如果滿足此條件，則輸出 "閏年"。
2.  接著，程式會獨立檢查第二個條件：年份 `Y` 是否能被 4 整除且不能被 100 整除 (`Y%4==0&&Y%100!=0`)。
    *   如果此條件為真，則輸出 "閏年"。
    *   如果此條件為假，則執行與第二個 `if` 語句相關聯的 `else` 區塊，輸出 "平年"。

值得注意的是，程式碼中的 `else` 語句只與它前面最近的一個 `if` 語句（即 `if(Y%4==0&&Y%100!=0)`）配對。這表示如果一個年份同時滿足第一個條件 (例如 2000 年)，它會先輸出一次 "閏年"。然後，由於第二個條件 (2000%4==0 && 2000%100!=0) 為假 (因為 2000%100 != 0 為假)，所以會再次進入第二個 `if` 的 `else` 區塊，輸出 "平年"。對於標準的閏年判斷，通常會使用 `if-else if-else` 結構或邏輯或 `||` 運算符來確保只有一種結果輸出。然而，此程式碼在 ZeroJudge 上獲得了 AC，這可能表示測試案例的設計或是判題系統的特定行為允許這種結構。

## 複雜度分析
- 時間複雜度: O(N)
  對於每個輸入的年份，程式執行固定次數的模運算 (modulo) 和比較運算。這些操作都是常數時間 O(1)。由於輸入包含 N 個年份直到 EOF，總體時間複雜度為 O(N)。
- 空間複雜度: O(1)
  程式只使用了一個整數變數 `Y` 來儲存當前處理的年份。無論輸入的年份數量有多少，所需的記憶體空間都是固定的常數空間。

## 程式碼
```cpp
#include <iostream>

using namespace std;

int main (){
	
	int Y;
	while (cin >> Y){
		if(Y%400==0){
			cout << "閏年" <<endl;
		}
		if(Y%4==0&&Y%100!=0){
			cout << "閏年" <<endl;
		}
		else{
			cout << "平年" <<endl; 
		}
	}
}

```

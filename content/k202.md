# k202 - 13055 - Inception

## 題目描述
這題模擬了電影《全面啟動》(Inception)中「夢中夢」的概念。Dom Cobb 可以進入不同人的夢境，一層一層地深入。有三種操作：
1.  **"Sleep X"**: Dom 進入名為 X 的人的夢境。這表示 X 的夢境會疊加在當前夢境之上。
2.  **"Kick"**: Dom 從當前夢境中被喚醒，回到上一層的夢境。如果 Dom 不在任何夢境中，此操作則無效。
3.  **"Test"**: Dom 查詢他目前身處在誰的夢境中。如果 Dom 不在任何夢境中，則輸出 "Not in a dream"。

程式需要處理 N 次查詢，並針對 "Test" 查詢輸出正確的結果。

## 解題思路
根據題目的描述，我們可以觀察到夢境的層次關係符合「後進先出」(LIFO) 的特性。當 Dom 進入一個新夢境時，它會疊加在現有夢境之上；當他被 "Kick" 驚醒時，他會從最深層的夢境回到上一層。這種行為模式非常適合使用堆疊 (Stack) 資料結構來實現。

具體操作如下：
1.  **"Sleep X"**: 當 Dom 進入 X 的夢境時，我們將 X 這個名字「推入」(push) 堆疊。
2.  **"Kick"**: 當 Dom 被喚醒時，我們需要回到上一層夢境。這表示將堆疊最頂端的元素「彈出」(pop)。在執行彈出操作前，需要檢查堆疊是否為空，如果為空則表示 Dom 不在任何夢境中，此時 "Kick" 操作無效。
3.  **"Test"**: 當 Dom 查詢他所在的夢境時，我們檢查堆疊是否為空。
    *   如果堆疊為空，表示 Dom 不在任何夢境中，輸出 "Not in a dream"。
    *   如果堆疊不為空，則 Dom 位於堆疊最頂端的那個人的夢境中，輸出堆疊頂端的元素 (top)。

程式碼中，使用了一個 `string` 型態的陣列 `a` 來模擬堆疊，並用 `sz` 變數作為堆疊的頂部指針（同時也代表堆疊中元素的數量）。
*   `a[sz++] = name;` 實現 push 操作。
*   `cout << a[sz-1] << "\n";` 實現 top 操作。
*   `--sz;` 實現 pop 操作。
*   `if(sz==0)` 實現檢查堆疊是否為空。

`ios::sync_with_stdio(0); cin.tie(0); cout.tie(0);` 這些語句用於加速 C++ 的輸入輸出，在競爭性程式設計中是常見的最佳化。

## 複雜度分析
- 時間複雜度: 每個操作 (Sleep, Kick, Test) 都只需要常數時間，即對陣列進行一次或兩次存取和更新 `sz` 變數。總共有 `n` 個查詢。因此，總的時間複雜度為 `O(n)`。
- 空間複雜度: 最壞情況下，所有的 `n` 個查詢都是 "Sleep"，這意味著我們需要儲存 `n` 個名字。每個名字的長度最大為 15。因此，所需的儲存空間與 `n` 成正比。空間複雜度為 `O(n * L)`，其中 `L` 是名字的最大長度。由於 `L` 是常數 (15)，因此可以簡化為 `O(n)`。

## 程式碼
```cpp
#include <iostream>
using namespace std;
int n,sz;
string a[10005],x;
int main(){
	cin.tie(0);cout.tie(0); ios::sync_with_stdio(0);
	cin >> n;
	while(n--){
		cin >> x;
		if(x=="Sleep"){
			cin >> a[sz++];
		}
		else if(x=="Test"){
			if(sz==0)cout << "Not in a dream\n";
			else cout << a[sz-1] << "\n";
		}
		else{
			if(sz)--sz;
		}
	}
} 
```

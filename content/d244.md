# d244 - 一堆石頭

## 題目描述
本題要求從一串給定的石頭編號中，找出哪一顆石頭被弄丟了。一開始，潘潘擁有一些石頭，每顆石頭都被複製了兩次，因此每個獨特的編號都會有三顆石頭。然而，她不小心掉了一顆石頭，導致現在總石頭數目為三的倍數減一個。輸入會給出她現在所擁有的所有石頭編號，用空格分隔，輸出遺失石頭的編號。

## 解題思路
這是一個經典的查找單一不同元素的問題變種。根據題目描述，每個獨特的石頭編號理論上應該出現三次。由於掉了一顆石頭，導致某個編號會只出現兩次，而其他編號仍然出現三次。

為了解決這個問題，我們可以採取以下步驟：
1.  **讀取輸入**: 由於輸入的所有石頭編號都在同一行，並用空格分隔，我們需要讀取整行字串。
2.  **解析數字**: 將讀取的字串透過 `stringstream` 解析成個別的整數，並儲存到一個陣列中。同時，計算出總共有多少顆石頭（即陣列的大小 `n`）。
3.  **排序**: 為了方便找出重複模式，將這個整數陣列進行升序排序。排序後，相同編號的石頭會彼此相鄰。
4.  **查找遺失石頭**: 由於除了遺失的石頭編號會出現兩次，其他編號都出現三次，排序後的陣列會呈現 `X, X, X, Y, Y, Y, ...` 的模式。而遺失的石頭會打破這個模式，例如 `A, A, B, B, B, ...` (A 是遺失的石頭) 或 `A, A, A, B, B, C, C, C, ...` (B 是遺失的石頭)。
    程式碼利用一個迴圈，每次跳三個位置 (`i += 3`) 來檢查。在每個步進中，它會檢查 `b[i]`, `b[i+1]`, 和 `b[i+2]`。
    *   如果 `b[i] != b[i+1]` (例如 `1, 2, 2, 2...`)，這表示 `b[i]` 是那個僅出現兩次的數字的其中一個 (因為排序過，如果 `b[i]`是遺失的，它會是 `X, X, Y...` 的形式，不會是 `X, Y, Y...`，除非 `X` 是唯一的，也就是單獨的那個)。在這種情況下，`b[i]` 就是遺失的石頭編號。
    *   如果 `b[i] == b[i+1]` 但 `b[i+1] != b[i+2]` (例如 `1, 1, 2, 2, 2...`)，這表示 `b[i]` 和 `b[i+1]` 組成了遺失的石頭的兩次出現，而 `b[i+2]` 則是下一個編號的石頭。此時，`b[i]` 就是遺失的石頭編號。
    這兩種情況都被 `b[i]!=b[i+1] || b[i+1]!=b[i+2]` 這個條件涵蓋。只要找到第一個不符合 `X, X, X` 模式的 `b[i]`，它就是答案。

這個方法依賴於排序後相同元素會聚集在一起的特性，並透過檢查每三個元素來找出不符合預期模式的數字。

## 複雜度分析
- 時間複雜度:
    *   讀取一行輸入字串的時間為 O(L)，其中 L 是輸入行的長度。
    *   使用 `stringstream` 解析字串並存入陣列，大約為 O(L) 或 O(N * D)，其中 N 是石頭數量，D 是平均數字位數。
    *   對包含 N 個元素的陣列進行排序，時間複雜度為 O(N log N)。
    *   遍歷排序後的陣列尋找遺失石頭，最壞情況下會遍歷整個陣列，時間複雜度為 O(N)。
    *   綜合來看，總時間複雜度主要由排序決定，為 O(N log N)。
- 空間複雜度:
    *   儲存輸入字串 `a` 需要 O(L) 空間。
    *   儲存所有石頭編號的整數陣列 `b` 需要 O(N) 空間。
    *   綜合來看，總空間複雜度為 O(N + L)，通常簡化為 O(N)，因為 L 會隨著 N 的增加而增加。

## 程式碼
```cpp
#include <iostream>
#include <string>
#include <sstream> 
#include <algorithm>
using namespace std;
int main(){
	string a;
	while(getline(cin,a)){
		int n=1;
		for(int i=0;i<a.length();i++){
			if(a[i]==' ')
				n++;
		}
		stringstream ss(a);
		int b[n];
		for(int i=0;i<n;i++)
			ss>>b[i];
		sort(b,b+n);
		for(int i=0;i<n-1;i+=3){
			if(b[i]!=b[i+1]||b[i+1]!=b[i+2]){
				cout << b[i] << endl;
				break;
			}
		}
	}
}
```

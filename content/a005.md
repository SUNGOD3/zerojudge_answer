# a005 - Eva 的回家作業

**🏷️ 標籤**: `Arithmetic Progression`, `Geometric Progression`

## 題目描述
本題要求根據給定的數列前四項，判斷該數列是等差數列還是等比數列，並計算出其第五項。題目明確指出所有數列只可能是等差或等比數列，且數列的前五項均為不大於 10^5 的自然數，等比數列的比值也保證為自然數。輸入的第一行是數列的數目 `t`，接下來的 `t` 行每行包含四個整數，代表數列的前四項。對於每個數列，程式需要輸出其前五項。

## 解題思路
本題的核心在於判斷數列類型並根據類型計算第五項。由於數列只可能是等差或等比，我們需要設計判斷條件來區分這兩種情況。

對於一個數列 `b, c, d, e`：
1.  **判斷是否為等比數列**:
    如果是一個等比數列，則其公比 `r` 應該滿足 `c = b * r`, `d = c * r`, `e = d * r`。由於題目約定公比為自然數，這表示 `c` 必須是 `b` 的倍數，`d` 必須是 `c` 的倍數，`e` 必須是 `d` 的倍數，且這些比值必須相等。
    因此，判斷條件為 `(d / c == e / d)` 且 `(e % d == 0)`。其中 `e % d == 0` 是為了確保 `e/d` 是一個整數（自然數），避免浮點數比較或整數除法截斷的問題。如果滿足此條件，公比即為 `d / c` (或 `e / d`)，第五項為 `e * (d / c)`。

2.  **判斷是否為等差數列**:
    如果是一個等差數列，則其公差 `diff` 應該滿足 `c = b + diff`, `d = c + diff`, `e = d + diff`。
    因此，判斷條件為 `(d - c == e - d)`。如果滿足此條件，公差即為 `d - c` (或 `e - d`)，第五項為 `e + (d - c)`。

程式的實作順序通常是先判斷等比數列，因為等比數列的條件（公比為自然數）相對更嚴格。如果滿足等比數列的條件，則直接計算並輸出。否則，再判斷是否為等差數列。

輸入處理方面，程式首先讀取一個整數 `a` (代表測試案例數量 `t`)，然後使用 `while(cin >> b >> c >> d >> e)` 迴圈來讀取後續的四個數。這個迴圈會一直執行，直到輸入流中不再有四個整數為止，有效地處理了所有 `t` 個測試案例。

## 複雜度分析
-   **時間複雜度**:
    程式對於每一個數列，僅執行常數次的算術運算（除法、取模、減法、加法）和比較。如果共有 `T` 個測試案例（數列），則總時間複雜度為 `O(T)`。題目中 `T` 的最大值為 20，因此運算量非常小。

-   **空間複雜度**:
    程式僅使用了幾個整數變數 `a, b, c, d, e` 來儲存輸入的數值和控制迴圈。這些變數所需的記憶體空間是固定的，不隨輸入數列的數量或數值大小而改變。因此，空間複雜度為 `O(1)`。

#include <iostream>

using namespace std;

int main (){
	
	int a;
	int b=0,c=0,d=0,e=0;
	
	cin >> a;
	
	while(cin >> b >> c >> d >> e){
	
		if(e/d==d/c&&e%d==0){
			cout << b << " " << c << " " << d << " " << e << " " << e*(d/c) <<endl;
		}
		else if((e-d)==(d-c)){
			cout << b << " " << c << " " << d << " " << e << " " << e+(d-c) <<endl;
		}
		
	}
		
}

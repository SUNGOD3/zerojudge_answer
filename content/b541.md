# b541 - 看到這樣的分布，網友們都驚呆了

## 題目描述
本題要求生成外觀數列 (Look-and-Say sequence) 的第 `n` 項。外觀數列從 `1` 開始，後續的每一項都是前一項的描述。例如：
*   `a1 = 1`
*   `a2 = 11` (讀作「1個1」)
*   `a3 = 21` (讀作「2個1」)
*   `a4 = 1211` (讀作「1個2，1個1」)
*   `a5 = 111221` (讀作「1個1，1個2，2個1」)
*   `a6 = 312211` (讀作「3個1，2個2，1個1」)

輸入包含多筆測試資料，每筆輸入一個整數 `n` (1 <= n <= 40)，要求輸出外觀數列的第 `n` 項。題目保證答案長度小於 10^5。

## 解題思路
外觀數列的生成規則是典型的遞歸定義：每一項都由前一項衍生而來。由於輸入的 `n` 最大為 40，且有多筆測試資料，這提示我們可以使用預先計算 (pre-computation) 的方式來優化。我們可以將從 `a1` 到 `a40`（甚至更多）的所有項都計算出來並儲存起來，然後在每次查詢時直接輸出對應的結果。

具體的生成邏輯如下：
1.  初始化 `ans[0]` 為 "1"，代表數列的第一項。
2.  使用一個迴圈從 `i = 0` 迭代到 `i = 40` (為了生成 `ans[1]` 到 `ans[41]`)。在每一次迭代中，我們將從 `ans[i]` 生成 `ans[i+1]`。
3.  對於 `ans[i]` 中的每一個字符，我們需要進行「讀數」操作：
    *   維護一個當前字符 `c` 和其連續出現的次數 `tmp`。
    *   遍歷 `ans[i]` 的所有字符。
    *   如果當前字符與前一個字符相同，則 `tmp` 加一。
    *   如果當前字符與前一個字符不同，則表示一個連續的數字串結束了。我們需要將 `tmp` (次數) 和 `c` (字符) 附加到新的字符串 `b` 中。然後重置 `c` 為當前字符，`tmp` 為 1。
    *   處理完 `ans[i]` 的所有字符後，務必將最後一個連續數字串的 `tmp` 和 `c` 附加到 `b` 中。
4.  將最終生成的字符串 `b` 存入 `ans[i+1]`。
5.  當所有項都預先計算完畢後，對於每次輸入的 `n`，直接輸出 `ans[n-1]` 即可（因為陣列索引從 0 開始，而題目項數從 1 開始）。

這個方法有效地利用了 `n` 的較小上限，避免了重複計算，使得每次查詢都是常數時間操作。

## 複雜度分析
- 時間複雜度:
    *   預先計算階段：外觀數列的長度會快速增長（近似 `(1.3)^N`）。最長項 `a_40` 的長度約為 33000。生成每一項 `a_k` 需要遍歷 `a_{k-1}`，其長度為 `L_{k-1}`。因此，預先計算所有項的時間複雜度是從 `L_1` 到 `L_40` 的總和，即 `O(sum_{k=1 to 40} L_k)`。由於 `L_k` 呈指數級增長，總時間主要由最後幾項的生成所支配，但總操作數仍在可接受範圍內。
    *   查詢階段：每次查詢僅需陣列索引，為 `O(1)`。
- 空間複雜度:
    *   我們需要儲存從 `a1` 到 `a41` (最多 `a40` 輸出，預留 `a41` 的空間) 的所有字符串。最長項的長度約為 33000 個字符。因此，空間複雜度是 `O(sum_{k=1 to 41} L_k)`，或者近似為 `O(N * L_max)`，其中 `N` 為最大項數 (41)，`L_max` 為最大項的長度。總共儲存的字符數約為 `41 * 33000`，約 1.3 MB，遠小於記憶體限制。

## 程式碼
```cpp
#include <iostream>
using namespace std;
string ans[42]={"1"};
int main(){
	for(int i=0;i<41;++i){
		string b;
		char c=ans[i][0];
		int tmp=0;
		for(int j=0,al=ans[i].length();j<al;++j){
			if(c!=ans[i][j]){
				b+=tmp+48;
				b+=c;
				c=ans[i][j];
				tmp=1;
			}
			else{
				++tmp;
			}
		}
		b+=tmp+48;
		b+=c;
		ans[i+1]=b;
	}
	int n;
	while(cin >> n)
		cout << ans[n-1] << "\n";
} 

```

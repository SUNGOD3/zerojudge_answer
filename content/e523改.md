# e523改 - Error

[🔗 前往 ZeroJudge 原題](https://zerojudge.tw/ShowProblem?problemid=e523改)

## 題目描述
題目要求讀取一系列整數，對於每個輸入的整數 `b`，在預先定義的斐波那契數列中查找 `b` 的位置。如果 `b` 在數列中，則輸出其位置（索引從 1 開始，即數列的第幾項）。如果 `b` 不在數列中，則輸出 "-1"。輸入以 0 結尾，表示輸入結束。

## 解題思路
程式碼預先定義了一個包含前 30 個斐波那契數的整數陣列 `d`。程式碼迴圈讀取輸入，對於每個輸入 `b`，使用一個 `do-while` 迴圈在 `d` 陣列中查找 `b`。如果找到 `b`，則輸出其索引加 1（因為陣列索引從 0 開始，題目要求從 1 開始）。如果未找到 `b`，則輸出 "-1"。輸入以 0 結尾，程式碼會持續讀取直到讀取到 0 為止。程式碼使用了優化的輸入輸出方式 `getchar_unlocked` 和 `putchar_unlocked`，以提高效率。

## 複雜度分析
- 時間複雜度: O(n*m)，其中 n 是輸入的數量，m 是斐波那契數列的長度 (在本例中為 30)。因為對於每個輸入，都需要遍歷斐波那契數列。
- 空間複雜度: O(1)，因為程式碼只使用了固定大小的陣列 `d` 和一些常數大小的變數。
